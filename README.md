Заняття 4 - CRUD
GET (Read)
Приклад: GET /students

Тіла запиту немає

Якшо все ок, у відповіді отримаємо статус 200 і колекцію

Якщо колекція порожня у відповіді отримаємо статус 200 і колекцію

Якщо у клієнта немає прав доступа до ресурса то отримаємо статус 401 і помилку

Приклад: GET /students/:studentId

Тіла запиту немає

Параметр у URL

Якшо все ок, у відповіді отримаємо статус 200 і документ

Якщо не знайшли документ з таким ID то отримаємо статус 404 і помилку

Якщо параметр не пройшов валідацію то отримаємо статус 400 і помилку

Якщо у клієнта немає прав доступа до ресурса то отримаємо статус 401 і помилку

POST (Create)
Приклад: POST /students
Тіло запиту (req.body) містить дані для створення нового ресурсу
Якшо все ок, у відповіді отримаємо статус 201 і документ
Якщо тіло запиту не пройшло валідацію то отримаємо статус 400 і помилку
Якщо у клієнта немає прав доступа до ресурса то отримаємо статус 401 і помилку
Метод Model.create
DELETE (Delete)
Приклад: DELETE /students/:studentId
Видалення ресурсу за ідентифікатором
Тіла запиту немає
Якшо все ок, у відповіді отримаємо статус 200 і те як зробить бекендщик (наприклад видалений документ)
Якщо не знайшли документ з таким ID то отримаємо статус 404 і помилку
Якщо параметр не пройшов валідацію то отримаємо статус 400 і помилку
Якщо у клієнта немає прав доступа до ресурса то отримаємо статус 401 і помилку
Метод Model.findOneAndDelete
PATCH (Update)
Приклад: PATCH /students/:studentId
Часткове оновлення ресурсу
Тіло запиту (req.body) містить дані для оновлення існуючего ресурсу
Тіло запиту містить лише ті поля, які потрібно оновити
Якщо ресурс існує, оновити його, в іншому випадку повернути 404
Якшо все ок, у відповіді отримаємо статус 200 і оновлений документ
Якщо не знайшли документ з таким ID то отримаємо статус 404 і помилку
Якщо параметр або тіло запиту не пройшли валідацію то отримаємо статус 400 і помилку
Якщо у клієнта немає прав доступа до ресурса то отримаємо статус 401 і помилку
Метод Model.findOneAndUpdate
PUT (Update/Create)
Повне оновлення ресурсу або створення нового, якщо ресурс не існує
Метод Model.findOneAndUpdate з опцією upsert: true
Якщо ресурс існує, оновити його, в іншому випадку, створити ресурс
Приклад: PUT /students/:studentId
Тіло запиту містить всі поля ресурсу, навіть якщо деякі з них не змінюються
PUT = PATCH + POST
