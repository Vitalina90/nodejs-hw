Сервер, веб-сервер, бекенд — у чому різниця?
Коли ми пишемо застосунок на Node.js і Express, ми створюємо серверну програму. Вона запускається на компʼютері або хмарному сервері, слухає певний порт, приймає HTTP-запити від клієнтів і повертає HTTP-відповіді. Саме цю серверну частину застосунку ми зазвичай називаємо бекендом.

Сервер як компʼютер
Сервер — це компʼютер або віртуальна машина, на якій запускаються програми, доступні іншим компʼютерам через мережу.

Фізичний / віртуальний сервер
  └── Операційна система
        └── Node.js runtime
              └── Наш Express-застосунок
У цьому сенсі сервер — це місце, де працює програма.

Наприклад, Render, Railway, VPS, AWS EC2 — це середовища, де можна запустити бекенд.

Сервер як програма
Слово сервер також може означати не компʼютер, а програму, яка приймає запити й повертає відповіді.

Наприклад:

Браузер → HTTP-запит → Серверна програма → HTTP-відповідь
У нашому випадку такою серверною програмою буде застосунок на Node.js + Express.

Тобто коли ми кажемо "Сервер повернув відповідь", ми часто маємо на увазі не залізо, а саме серверну програму.

Веб-сервер
Веб-сервер — це програма, яка приймає HTTP-запити і повертає HTTP-відповіді.

Наприклад, веб-сервер може:

віддати HTML, CSS, JS-файли;
повернути JSON;
прийняти дані з форми;
обробити запит до API.
Класичні приклади веб-серверів це Nginx та Apache. Але Node.js-застосунок теж може працювати як веб-сервер, бо він може слухати порт і відповідати на HTTP-запити.

Backend
Backend — це серверна частина застосунку, яка працює не в браузері користувача, а на сервері.

Backend зазвичай відповідає за:

отримання запитів
обробку бізнес-логіки
роботу з базою даних
автентифікацію
повернення відповіді клієнту
Наприклад, коли фронтенд робить запит GET /api/notes

бекенд:

приймає запит
перевіряє користувача
дістає нотатки з бази даних
повертає JSON
Короткі визначення
Клієнт — це програма, яка надсилає запит до сервера. Наприклад: браузер, фронтенд-застосунок, мобільний застосунок або Postman.

Сервер — це або компʼютер, на якому працює програма, або сама програма, яка приймає запити й повертає відповіді.

Веб-сервер — це програма, яка працює з HTTP-запитами та HTTP-відповідями.

Backend — це серверна частина застосунку: код, який обробляє запити, працює з базою даних, виконує логіку й повертає дані клієнту.
